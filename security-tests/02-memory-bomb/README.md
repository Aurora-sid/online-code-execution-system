# 内存炸弹攻击测试

## 攻击原理
内存炸弹通过不断申请和使用大量内存，耗尽系统可用内存，导致系统OOM（Out of Memory）或严重性能下降。

## 测试用例

### test.c (C语言)
- 使用 `malloc()` 申请1GB内存块
- 使用 `memset()` 实际写入内存（确保内存真正被使用）

### test.cpp (C++)
- 使用 `new` 运算符申请100MB内存块
- 使用 `std::vector` 存储指针防止被优化掉

### test.py (Python)
- 创建大字符串（每次100MB）
- 存储在列表中防止垃圾回收

### test.java (Java)
- 申请大字节数组
- 实际填充数据确保JVM真正分配内存

## 预期安全响应

✅ **正确行为**：
- 错误信息：`执行超时：内存使用过高(>90%)，可能触发了内存炸弹攻击`
- 或：`执行超时：内存耗尽被系统终止(OOM Killed)`
- 容器被强制停止
- 不影响宿主机内存

❌ **失败情况**：
- 成功分配超过512MB内存
- 宿主机内存异常增长
- 容器未被终止

## 安全配置验证

- `Memory: 512MB` - 限制容器最大内存使用
- `MemorySwap: 512MB` - 禁用swap防止swap耗尽
- `diagnoseTimeout()` - 检测 `MemoryStats.Usage > 90%` 或 `OOMKilled == true`
