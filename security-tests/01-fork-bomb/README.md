# Fork炸弹攻击测试

## 攻击原理
Fork炸弹通过不断创建新进程，快速耗尽系统进程资源，导致系统无法创建新进程而崩溃。

## 测试用例

### test.c (C语言)
- 使用 `fork()` 系统调用无限循环创建进程
- 最高效的实现方式

### test.py (Python)
- 使用 `os.fork()` 创建子进程
- 在支持的Unix系统上有效

### test.java (Java)
- 使用 `ProcessBuilder` 创建新进程
- 模拟进程耗尽攻击

## 预期安全响应

✅ **正确行为**：
- 错误信息：`执行超时：进程数量达到限制(50)，可能触发了 fork 炸弹攻击`
- 容器被强制停止并清理
- 不影响宿主机系统

❌ **失败情况**：
- 代码成功执行且无错误
- 容器未被停止
- 宿主机进程数异常增长

## 安全配置验证

- `PidsLimit: 50` - 限制容器最多创建50个进程
- 超时机制 - 5秒后强制终止
- `diagnoseTimeout()` - 检测 `PidsStats.Current >= 50`
