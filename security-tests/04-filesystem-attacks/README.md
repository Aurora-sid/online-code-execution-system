# 文件系统攻击测试

## 攻击原理
攻击者尝试读取敏感系统文件（如 `/etc/passwd`）获取用户信息，或者尝试写入系统目录植入恶意软件，或者写入超大文件耗尽磁盘空间。

## 测试用例

### read-passwd.py
- 尝试读取 `/etc/passwd`
- **预期**：只能读取到容器内部的 `/etc/passwd`，内容应很少且无敏感宿主信息。

### write-system.py
- 尝试写入 `/etc/` 和 `/usr/bin/` 等系统目录
- **预期**：抛出 `Permission denied` 错误（如果不是 root 运行）或只能在容器层写入（如果容器未配置只读根文件系统）。虽然容器内是临时写层，但验证无法写入宿主机文件系统是核心。

### disk-bomb.py
- 尝试在工作目录写入 1GB 文件
- **预期**：如果配置了容器存储限制，写入应失败；否则可能会成功写入容器临时层（需注意宿主机磁盘空间）。

## 安全配置验证

- 文件系统隔离：确保无法挂载或访问宿主机文件系统
- 权限控制：推荐使用非 root 用户运行代码（目前 Sandbox可能默认 root，需验证）
- 存储限制：`StorageOpt` 配置（如果已启用）
