# CPU耗尽攻击测试

## 攻击原理
CPU耗尽攻击通过执行无限循环或极高计算量的任务，长期占用CPU资源，导致其他进程无法获得CPU时间片，从造成拒绝服务。

## 测试用例

### test.c (C语言)
- 简单的 `while(1)` 死循环

### test.py (Python)
- Python层面的死循环

### test.go (Go)
- Go语言的死循环

## 预期安全响应

✅ **正确行为**：
- 错误信息：`执行超时：代码运行时间超过5秒限制，可能是死循环或计算量过大`
- 容器被强制停止
- 宿主机CPU使用率受控（`NanoCPUs: 0.5` 限制）

❌ **失败情况**：
- 代码运行超过5秒未停止
- 宿主机CPU使用率飙升到100%
- 多个容器同时运行时导致系统卡顿

## 安全配置验证

- `NanoCPUs: 500000000` (0.5 CPU) - 硬性限制CPU使用率
- `Context Timeout: 5s` - 应用层超时控制
- `ContainerStop` - 超时后强制清理
