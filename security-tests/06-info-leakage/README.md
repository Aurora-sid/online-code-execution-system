# 信息泄露测试

## 攻击原理
攻击者尝试获取宿主机的敏感配置信息（如环境变量中的 API Key、数据库密码）或探测宿主机运行的其他进程信息。

## 测试用例

### env-vars.py
- 枚举所有环境变量
- 检查是否包含敏感关键字（KEY, SECRET, PASSWORD等）
- **预期**：只显示容器默认的环境变量（如 `PATH`, `HOSTNAME` 等），不包含宿主机敏感信息。

### proc-info.py
- 读取 `/proc` 目录枚举进程ID
- **预期**：只能看到容器内的少数几个进程（通常少于10个），证明 PID Namespace 隔离生效。如果能看到大量 PID（如 > 100），说明隔离可能失败。

## 安全配置验证

- 环境变量隔离：Docker 默认隔离
- PID Namespace：Docker 默认启用
- 避免使用 `--privileged` 标志启动容器
- 避免挂载宿主机 `/proc` 到容器
